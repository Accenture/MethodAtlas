MethodAtlas is a command-line tool and library for scanning test source trees, extracting structured metadata about test methods, and optionally enriching that metadata with AI-generated security classification suggestions. Supported languages include Java (JUnit 4/5, TestNG) and C# (xUnit, NUnit, MSTest) via a plugin architecture.

The tool parses source files without compiling them, which allows it to run against any project regardless of build setup. For each discovered test method it reports the fully qualified class name, method name, inclusive line count, and any tag or annotation values already present in the source. When AI enrichment is enabled, it also produces suggested security tags, a human-readable display name, an explanatory rationale, and an optional confidence score.

Key Capabilities

  • Source-level test inventory — discovers test methods in Java (JUnit 4/5, TestNG) and C# (xUnit, NUnit, MSTest) without symbol resolution or compilation, using a plugin-based discovery architecture.
  • Multiple output formats — CSV (default), plain text, and SARIF 2.1.0 for direct integration with GitHub Code Scanning and other SAST tooling.
  • AI-assisted security classification — submits each test class to a configurable AI provider (Ollama, OpenAI, OpenRouter, Anthropic, or auto-detected) and returns per-method security relevance, taxonomy tags, and reasoning.
  • Source write-back — the {@code -apply-tags} mode inserts AI-generated {@code @DisplayName} and {@code @Tag} annotations directly into the source files using a lexical-preserving printer.
  • Manual AI workflow — a two-phase prepare/consume mode for operators who interact with an AI through a chat interface rather than a direct API.
  • Content fingerprinting — optional SHA-256 content hash per class enables incremental scanning and result traceability across runs.
  • YAML configuration — scan settings can be stored in a shared configuration file and overridden selectively on the command line.

Package Structure

{@link org.egothor.methodatlas}
Core application layer. Contains the main entry point ({@link org.egothor.methodatlas.MethodAtlasApp}), command-line argument parsing ({@link org.egothor.methodatlas.CliArgs}, {@link org.egothor.methodatlas.CliConfig}), and cross-cutting value types ({@link org.egothor.methodatlas.OutputMode}, {@link org.egothor.methodatlas.TagAiDrift}, {@link org.egothor.methodatlas.DeltaEntry}, {@link org.egothor.methodatlas.DeltaReport}).
{@link org.egothor.methodatlas.api}
Platform-neutral API contracts. Defines the discovery SPI ({@link org.egothor.methodatlas.api.TestDiscovery}, {@link org.egothor.methodatlas.api.TestDiscoveryConfig}, {@link org.egothor.methodatlas.api.DiscoveredMethod}, {@link org.egothor.methodatlas.api.SourceContent}, {@link org.egothor.methodatlas.api.SourcePatcher}) and the scan record ({@link org.egothor.methodatlas.api.ScanRecord}).
{@code org.egothor.methodatlas.discovery.jvm}
Java/JVM test discovery implementation (module {@code methodatlas-discovery-jvm}). Walks source trees with JavaParser and emits {@link org.egothor.methodatlas.api.DiscoveredMethod} records ({@code JavaTestDiscovery}); annotation inspection utilities are in {@code AnnotationInspector}. Supports JUnit 4/5 and TestNG via automatic framework detection.
{@code org.egothor.methodatlas.discovery.dotnet}
C#/.NET test discovery implementation (module {@code methodatlas-discovery-dotnet}). Parses {@code .cs} source files with an ANTLR-based grammar and detects test attributes for xUnit, NUnit, and MSTest ({@code DotNetTestDiscovery}). Also provides source write-back via {@code DotNetSourcePatcher}.
{@link org.egothor.methodatlas.emit}
Output emitters. Formats enriched records as CSV or plain text ({@link org.egothor.methodatlas.emit.OutputEmitter}), SARIF 2.1.0 ({@link org.egothor.methodatlas.emit.SarifEmitter}), GitHub Actions workflow commands ({@link org.egothor.methodatlas.emit.GitHubAnnotationsEmitter}), and delta reports ({@link org.egothor.methodatlas.emit.DeltaEmitter}).
{@link org.egothor.methodatlas.ai}
AI integration layer. Defines the provider-agnostic {@link org.egothor.methodatlas.ai.AiSuggestionEngine} contract, provider client implementations ({@link org.egothor.methodatlas.ai.OllamaClient}, {@link org.egothor.methodatlas.ai.OpenAiCompatibleClient}, {@link org.egothor.methodatlas.ai.AnthropicClient}), the controlled security taxonomy ({@link org.egothor.methodatlas.ai.DefaultSecurityTaxonomy}, {@link org.egothor.methodatlas.ai.OptimizedSecurityTaxonomy}), prompt construction ({@link org.egothor.methodatlas.ai.PromptBuilder}), and the manual workflow engines ({@link org.egothor.methodatlas.ai.ManualPrepareEngine}, {@link org.egothor.methodatlas.ai.ManualConsumeEngine}).
