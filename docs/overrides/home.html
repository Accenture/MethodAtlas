MethodAtlas

Scan Java test sources, extract JUnit 5 metadata, and enrich results with AI-generated security classification — without compiling a single class.

Get started CLI Reference
📋

Source-level test inventory

Discovers @Test, @ParameterizedTest, @RepeatedTest, @TestFactory, and @TestTemplate methods using a lexical parser — no compilation required.

🤖

AI security classification

Submits each test class to Ollama, OpenAI, OpenRouter, or Anthropic and returns per-method security relevance, taxonomy tags, and human-readable rationale.

📄

Multiple output formats

Exports results as CSV (default), plain text, or SARIF 2.1.0 for direct integration with GitHub Code Scanning and other SAST tooling.

✏️

Source write-back

The -apply-tags mode inserts AI-generated @DisplayName and @Tag annotations directly into source files using a lexical-preserving printer.

💬

Manual AI workflow

Two-phase prepare / consume mode for operators who interact with an AI model through a chat interface rather than a direct API connection.

⚙️

YAML configuration

Scan settings are stored in a shareable configuration file and can be overridden selectively on the command line — no flags required for repeated runs.

Quick start

# Scan a project and write CSV to stdout
java -jar methodatlas.jar -scan /path/to/project/src/test/java

# Enrich with local Ollama and save to file
java -jar methodatlas.jar \
  -scan /path/to/project/src/test/java \
  -ai-provider ollama \
  -ai-model llama3 \
  -output results.csv

# SARIF output for GitHub Code Scanning
java -jar methodatlas.jar \
  -scan /path/to/project/src/test/java \
  -format sarif \
  -output results.sarif
